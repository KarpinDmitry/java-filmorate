## Filmorate

**Filmorate** — это backend-приложение для управления каталогом фильмов и взаимодействия пользователей с ними. Сервис позволяет хранить информацию о фильмах, регистрировать пользователей, ставить лайки и формировать рейтинг самых популярных фильмов.

Проект реализован как REST API и демонстрирует построение серверного приложения на Java с использованием современных принципов разработки и архитектуры.

### Основные возможности

* регистрация и управление пользователями
* добавление и редактирование фильмов
* система лайков для фильмов
* формирование списка **самых популярных фильмов** на основе количества лайков
* получение списка фильмов и пользователей через REST API
* валидация входных данных и обработка ошибок

### Архитектура

Проект построен по многослойной архитектуре:

* **Controller layer** — обработка HTTP-запросов и работа с REST API
* **Service layer** — бизнес-логика приложения
* **Storage layer** — работа с данными (хранение пользователей и фильмов)

Такое разделение позволяет упростить тестирование, поддержку и масштабирование приложения.

### Технологии

* **Java**
* **Spring Boot**
* **REST API**
* **Maven**
* **JUnit**
* **PostgreSQl**
* **SpringData**

### Цель проекта

Проект разработан как учебный backend-сервис и демонстрирует навыки:

* проектирования REST API
* работы с многослойной архитектурой
* обработки пользовательских данных
* реализации бизнес-логики и рейтинговых алгоритмов

Filmorate может служить основой для более сложных сервисов рекомендаций фильмов или социальных платформ для киноманов.

### Описание схемы

База данных состоит из следующих таблиц:

- `films` — фильмы.
- `genres` — жанры.
- `mpa` — возрастные рейтинги (MPA).
- `film_genres` — связь между фильмами и жанрами (многие ко многим).
- `users` — пользователи.
- `friendships` — дружба между пользователями (самосвязь).
- `likes` — лайки пользователей 

#### Основные связи:
- Один фильм может иметь несколько жанров → через таблицу `film_genres`.
- Один жанр может быть у нескольких фильмов.
- Фильм может не иметь рейтинга MPA (опциональная связь).
- Пользователь может быть другом другого пользователя → таблица `friendships`.
